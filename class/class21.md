## 1. 텍스트 파일 입출력

C에서 파일을 다루려면 `<stdio.h>`에 있는 파일 입출력 함수를 사용합니다. 파일을 **열고(`fopen`)** → **읽거나 쓰고** → **닫는(`fclose`)** 3단계가 기본입니다.

### 1.1 파일에 쓰고, 한 줄 읽기 (fgets + sscanf_s)

```c
#include <stdio.h>
#include <string.h>

int main()
{
	FILE* pf = NULL;

	fopen_s(&pf, "test.txt", "wt");
	fprintf(pf, "10 2.5 H ABC\n");
	fclose(pf);

	fopen_s(&pf, "test.txt", "rt");
	char buf[100];
	fgets(buf, sizeof(buf), pf);
	printf("size: %d, %s", strlen(buf), buf);
	fclose(pf);

	int n;
	double d;
	char c;
	char s[100];
	sscanf_s(buf, "%d %lf %c %s", &n, &d, &c, 1, s, 100);
	printf("%d\n", n);
	printf("%g\n", d);
	printf("%c\n", c);
	printf("%s\n", s);
}
```

**설명**

- `FILE* pf = NULL;` : 파일을 가리키는 **파일 포인터**를 선언합니다.
- **파일 쓰기 단계**
  - `fopen_s(&pf, "test.txt", "wt")` : `"test.txt"` 파일을 **텍스트 쓰기 모드**로 엽니다.
    - `"wt"` : **w**(write, 쓰기) + **t**(text, 텍스트 모드). 파일이 없으면 새로 생성하고, 있으면 내용을 지우고 처음부터 씁니다.
  - `fprintf(pf, "10 2.5 H ABC\n")` : 파일에 문자열을 씁니다. `printf`와 형식이 같지만 첫 번째 인자로 **파일 포인터**를 받습니다.
  - `fclose(pf)` : 파일을 닫습니다. 열었으면 **반드시 닫아야** 합니다.
- **파일 읽기 단계**
  - `fopen_s(&pf, "test.txt", "rt")` : 같은 파일을 **텍스트 읽기 모드**(`"rt"`)로 다시 엽니다.
  - `fgets(buf, sizeof(buf), pf)` : 파일에서 **한 줄**을 읽어 문자 배열 `buf`에 저장합니다.
  - `strlen(buf)` : 문자열의 길이를 반환합니다. (`<string.h>` 필요)
- **문자열 파싱 단계**
  - `sscanf_s(buf, "%d %lf %c %s", &n, &d, &c, 1, s, 100)` : **문자열(`buf`)** 에서 형식 지정자에 맞게 값을 추출합니다.
    - `sscanf_s`는 `scanf_s`의 **문자열 버전**으로, 키보드 대신 문자열에서 읽습니다.
    - `%c` 뒤의 `1`과 `%s` 뒤의 `100`은 MSVC의 보안 함수에서 요구하는 **버퍼 크기** 인자입니다.

**예상 출력**
```
size: 15, 10 2.5 H ABC
10
2.5
H
ABC
```

### 1.2 파일에 쓰고, 형식 지정자로 직접 읽기 (fscanf_s)

```c
#include <stdio.h>
#include <string.h>

int main()
{
	FILE* pf = NULL;

	fopen_s(&pf, "test.txt", "wt");
	fprintf(pf, "10 2.5 H ABC\n");
	fclose(pf);

	int n;
	double d;
	char c;
	char s[100];

	fopen_s(&pf, "test.txt", "rt");
	fscanf_s(pf, "%d %lf %c %s", &n, &d, &c, 1, s, 100);
	fclose(pf);

	printf("%d\n", n);
	printf("%g\n", d);
	printf("%c\n", c);
	printf("%s\n", s);
}
```

**설명**

- 1.1과 같은 내용을 파일에 쓴 뒤, `fscanf_s`로 **파일에서 직접** 형식 지정자에 맞게 값을 읽습니다.
- `fscanf_s(pf, "%d %lf %c %s", ...)` : `scanf_s`와 사용법이 같지만, 키보드 대신 **파일 포인터(`pf`)**에서 읽습니다.
- **1.1 방식(fgets + sscanf_s)과의 비교**

| 방식 | 장점 | 단점 |
|------|------|------|
| `fgets` + `sscanf_s` | 한 줄을 먼저 읽어 안전하게 처리 가능 | 코드가 두 단계로 나뉨 |
| `fscanf_s` 직접 | 코드가 간결함 | 줄 단위 제어가 어려움 |

**예상 출력**
```
10
2.5
H
ABC
```

## 2. 전처리기 지시문 (#define)

`#define`은 **컴파일 전**에 처리되는 **전처리기 지시문**입니다. 코드에서 이름을 값이나 식으로 **텍스트 치환**합니다.

### 2.1 상수 매크로와 함수형 매크로 기초

```c
#include <stdio.h>

#define MAX		100
#define Add(a, b) (a+b)

int main()
{
	printf("%d\n", MAX);
	printf("%d\n", 100);
	printf("add : %d\n", Add(2, 3));
	printf("add : %d\n", (2+3));
}
```

**설명**

- `#define MAX 100` : **상수 매크로**. 코드에서 `MAX`가 나오면 컴파일 전에 모두 `100`으로 치환됩니다.
  - `printf("%d\n", MAX)` → 실제로는 `printf("%d\n", 100)`으로 바뀌어 컴파일됩니다.
  - 매직 넘버(의미가 불분명한 숫자)를 이름으로 대체하여 **가독성**을 높이는 데 사용합니다.
- `#define Add(a, b) (a+b)` : **함수형 매크로**. `Add(2, 3)` → `(2+3)`으로 치환됩니다.
  - 함수 호출이 아닌 **텍스트 치환**이므로, 실행 시간에 호출 비용이 없습니다.

**예상 출력**
```
100
100
add : 5
add : 5
```

---

### 2.2 매크로와 실제 함수 비교

```c
#include <stdio.h>

#define MAX		100
#define Add(a, b) (a+b)
int Adder(int a, int b)
{
	return a + b;
}

int main()
{
	printf("%d\n", MAX);
	printf("%d\n", 100);
	printf("add : %d\n", Add(2, 3));
	printf("add : %d\n", (2 + 3));
}
```

**설명**

- `int Adder(int a, int b)` : **실제 함수**로 덧셈을 구현한 예입니다.
- 매크로 `Add`와 함수 `Adder`는 같은 결과를 내지만, 동작 방식이 다릅니다.

| 비교 항목 | 매크로 (`#define`) | 함수 |
|-----------|-------------------|------|
| **동작 원리** | 텍스트 치환 (전처리 단계) | 호출 시 인자 계산 후 전달 (실행 단계) |
| **타입 검사** | 없음 | 있음 (컴파일러가 검사) |
| **디버깅** | 어렵다 (치환 후 코드만 보임) | 쉽다 (브레이크포인트 설정 가능) |
| **호출 비용** | 없음 | 함수 호출 오버헤드 있음 |

---

### 2.3 함수형 매크로의 괄호 문제 — 잘못된 예

```c
#include <stdio.h>

#define MAX		100
#define Mul(a, b) (a*b)
int Multi(int a, int b)
{
	return a * b;
}

int main()
{
	int n1 = 2;
	int n2 = 3;

	printf("%d\n", Mul(n1 + n2, n1 + n2));
	printf("%d\n", (n1 + n2 * n1 + n2));
	printf("%d\n", Multi(n1 + n2, n1 + n2));
}
```

**설명**

- `#define Mul(a, b) (a*b)` : 매크로의 인자에 **괄호가 없는** 잘못된 예입니다.
- `Mul(n1 + n2, n1 + n2)` 가 치환되면:
  1. `(n1 + n2 * n1 + n2)` ← 텍스트 그대로 치환
  2. `(2 + 3 * 2 + 3)` ← 값 대입
  3. `(2 + 6 + 3)` = **11** ← `*`이 `+`보다 우선순위가 높음
  - 의도한 값은 `(2+3) * (2+3)` = **25**이지만, **연산자 우선순위** 때문에 전혀 다른 결과가 나옵니다.
- 두 번째 `printf`는 치환 결과와 동일한 식을 직접 작성한 것으로, 같은 **11**이 나옵니다.
- `Multi(n1 + n2, n1 + n2)` : **함수**는 인자를 **먼저 계산**한 뒤 전달하므로 `Multi(5, 5)` = **25**가 정상적으로 나옵니다.

**예상 출력**
```
11
11
25
```

---

### 2.4 함수형 매크로의 괄호 문제 — 올바른 예

```c
#include <stdio.h>

#define MAX		100
#define Mul(a, b) ((a)*(b))
int Multi(int a, int b)
{
	return a * b;
}

int main()
{
	int n1 = 2;
	int n2 = 3;

	printf("%d\n", Mul(n1 + n2, n1 + n2));
	printf("%d\n", (n1 + n2 * n1 + n2));
	printf("%d\n", Multi(n1 + n2, n1 + n2));
}
```

**설명**

- `#define Mul(a, b) ((a)*(b))` : 인자마다 **괄호**로 감싸서 연산자 우선순위 문제를 해결합니다.
- `Mul(n1 + n2, n1 + n2)` 가 치환되면:
  1. `((n1 + n2)*(n1 + n2))` ← 괄호가 있어 올바르게 묶임
  2. `((2 + 3)*(2 + 3))` = `5 * 5` = **25**
- 두 번째 `printf`는 매크로가 아닌 **직접 작성한 식** `(n1 + n2 * n1 + n2)`이므로 여전히 **11**입니다.
- **핵심 규칙**: 함수형 매크로를 정의할 때는 **반드시 각 인자와 전체 식을 괄호로 감싸야** 합니다.
  - 나쁜 예: `#define Mul(a, b) (a*b)`
  - 좋은 예: `#define Mul(a, b) ((a)*(b))`

**예상 출력**
```
25
11
25
```

## 3. 삼항 연산자와 Max 함수

**삼항 연산자**(`? :`)는 `if-else`를 한 줄로 줄여 쓸 수 있는 연산자입니다.

### 3.1 삼항 연산자 기본과 Max 함수

```c
#include <stdio.h>

int Max(int a, int b)
{
	return a > b ? a : b;
}
int main()
{
	int n1 = 10;
	int n2 = 20;

	printf("%d\n", Max(n1, n2));

	int result = n1 > n2 ? n1 : n2;
	printf("%d\n", result);

	n1 < n2 ? printf("Good!\n") : printf("Bad!\n");
}
```

**설명**

- **삼항 연산자 형식**: `조건 ? 참일_때_값 : 거짓일_때_값`
  - `a > b ? a : b` : `a`가 `b`보다 크면 `a`를 반환, 아니면 `b`를 반환 → **두 수 중 큰 값**
- `Max(n1, n2)` : 함수 안에서 삼항 연산자로 큰 값을 반환합니다. `Max(10, 20)` → **20**
- `int result = n1 > n2 ? n1 : n2;` : 삼항 연산자를 **변수 대입**에 직접 사용한 예. `10 > 20`은 거짓이므로 `result = 20`
- `n1 < n2 ? printf("Good!\n") : printf("Bad!\n");` : 삼항 연산자의 참/거짓 자리에 **함수 호출**을 넣을 수도 있습니다. `10 < 20`은 참이므로 `"Good!"`이 출력됩니다.

**예상 출력**
```
20
20
Good!
```

---

### 3.2 #define Max 매크로와 증감 연산자의 부작용

```c
#include <stdio.h>

//int Max(int a, int b)
//{
//	return a > b ? a : b;
//}
#define Max(a, b) (((a)>(b)) ? (a) : (b))
int main()
{
	int n1 = 10;
	int n2 = 20;

	printf("%d\n", Max(n1, n2));
	printf("%d\n", Max(++n1, ++n2));
	printf("%d, %d\n", n1, n2);
}
```

**설명**

- `#define Max(a, b) (((a)>(b)) ? (a) : (b))` : `Max`를 **매크로**로 구현한 예입니다. 주석 처리된 함수 버전과 비교해 보세요.
- `Max(n1, n2)` → `(((10)>(20)) ? (10) : (20))` → **20**. 여기까지는 정상입니다.
- **문제 발생**: `Max(++n1, ++n2)` → `(((++n1)>(++n2)) ? (++n1) : (++n2))`
  - `++n1`과 `++n2`가 식에 **두 번씩** 나타나므로, 증감이 한 번이 아닌 **여러 번** 일어날 수 있습니다.
  - 이는 **정의되지 않은 동작(undefined behavior)**에 해당하며, 컴파일러마다 결과가 다를 수 있습니다.
- **교훈**: 매크로에는 `++`, `--` 같은 **부작용이 있는 인자**를 넣으면 안 됩니다. 이런 경우 주석 처리된 것처럼 **함수**를 사용하는 것이 안전합니다.

## 4. 조건부 컴파일

### 4.1 #ifdef / #else / #endif

```c
#include <stdio.h>
//#define _WINDOWS_

int main()
{
#ifdef _WINDOWS_
	printf("windows OS Logo!\n");
#else
	printf("linux OS Logo!\n");
#endif

	int n = 100;

	printf("int : %d\n", n);
}
```

**설명**

- `#ifdef _WINDOWS_` : `_WINDOWS_`가 `#define`으로 **정의되어 있으면** 바로 아래 코드를 컴파일에 포함합니다.
- `#else` : 정의되어 있지 **않으면** 이 부분을 컴파일에 포함합니다.
- `#endif` : 조건부 컴파일 블록의 **끝**을 표시합니다.
- 위 예제에서 `//#define _WINDOWS_`는 **주석 처리**되어 있으므로, `_WINDOWS_`가 정의되지 않은 상태입니다. 따라서 `#else` 쪽의 `"linux OS Logo!"`가 출력됩니다.
- 주석을 해제하면(`#define _WINDOWS_`로 변경) `"windows OS Logo!"`가 출력됩니다.
- `#ifdef`/`#endif` 바깥의 코드(`int n = 100;` 이하)는 **조건에 관계없이 항상** 컴파일됩니다.

**활용 사례**

| 매크로 정의 | 출력 |
|------------|------|
| `//#define _WINDOWS_` (주석 처리 = 미정의) | `linux OS Logo!` |
| `#define _WINDOWS_` (정의됨) | `windows OS Logo!` |

- 실무에서는 운영체제, **디버그/릴리즈** 모드, 플랫폼 차이 등에 따라 **다른 코드를 선택적으로 컴파일**할 때 사용합니다.

**예상 출력 (`_WINDOWS_` 미정의 시)**
```
linux OS Logo!
int : 100
```

## 5. 구조체 (struct)

**구조체**는 서로 관련 있는 여러 변수를 **하나로 묶어서** 새로운 타입을 만드는 것입니다.

### 5.1 구조체 정의, 함수, 배열 활용

```c
#include <stdio.h>
struct Data
{
	int n;
	double d;
};
void PrintData(const Data* pd) {
	printf("%d, %g\n", pd->n, pd->d);
}
void SetData(Data* pd, int n, double d) {
	pd->n = n;
	pd->d = d;
}
int main()
{
	Data darr[5] = { 0 };
	int arr[5] = { 10,20,30,40,50 };

	for (int i = 0; i < 5; ++i)
		SetData(&darr[i], i + 1, i + 10);

	for (int i = 0; i < 5; ++i)
		PrintData(&darr[i]);

	int* pa = arr;
	for (int i = 0; i < 5; ++i)
	{
		printf("int [%d] : %d\n", i, arr[i]);
	}
}
```

**설명**

- **구조체 정의**: `struct Data { int n; double d; };`
  - 정수 `n`과 실수 `d`를 **하나의 타입**으로 묶었습니다.
  - `Data`라는 이름으로 새 타입처럼 사용할 수 있습니다.
- **구조체 포인터를 이용한 함수**
  - `void PrintData(const Data* pd)` : 구조체 **포인터**를 매개변수로 받아 멤버를 출력합니다.
    - `pd->n` : 포인터로 구조체 멤버에 접근할 때 **화살표 연산자(`->`)**를 사용합니다.
    - `const` : 함수 안에서 구조체 내용을 **수정하지 않겠다**는 약속입니다.
  - `void SetData(Data* pd, int n, double d)` : 구조체 포인터를 받아 멤버 값을 **설정**합니다.
    - `const`가 없으므로 포인터를 통해 값을 변경할 수 있습니다.
- **구조체 배열**: `Data darr[5] = { 0 };`
  - `Data` 타입 배열 5개를 선언하고 **0으로 초기화**합니다.
  - 반복문에서 `SetData(&darr[i], i + 1, i + 10)`으로 각 요소에 값을 설정합니다.

| 인덱스 | `darr[i].n` | `darr[i].d` |
|--------|-------------|-------------|
| 0 | 1 | 10 |
| 1 | 2 | 11 |
| 2 | 3 | 12 |
| 3 | 4 | 13 |
| 4 | 5 | 14 |

- **배열과 포인터**: `int* pa = arr;`
  - 배열 이름(`arr`)은 **첫 번째 요소의 주소**와 같으므로 포인터 변수에 대입할 수 있습니다.

**예상 출력**
```
1, 10
2, 11
3, 12
4, 13
5, 14
int [0] : 10
int [1] : 20
int [2] : 30
int [3] : 40
int [4] : 50
```

## 6. 보충 학습

### 6.1 파일 모드 정리

`fopen_s`의 두 번째 인자로 지정하는 **파일 열기 모드**입니다.

| 모드 | 의미 | 파일 없을 때 | 파일 있을 때 |
|------|------|-------------|-------------|
| `"rt"` | 텍스트 읽기 | 오류 | 처음부터 읽기 |
| `"wt"` | 텍스트 쓰기 | 새로 생성 | 내용 삭제 후 쓰기 |
| `"at"` | 텍스트 추가 | 새로 생성 | 끝에 이어 쓰기 |
| `"rb"` | 바이너리 읽기 | 오류 | 처음부터 읽기 |
| `"wb"` | 바이너리 쓰기 | 새로 생성 | 내용 삭제 후 쓰기 |

### 6.2 fopen_s vs fopen

- `fopen_s`는 **MSVC(Visual Studio)** 전용 보안 함수입니다. 반환값 대신 첫 번째 인자로 파일 포인터를 받습니다.
- 표준 C의 `fopen`은 다음과 같이 사용합니다:
  ```c
  FILE* pf = fopen("test.txt", "wt");
  ```
- MSVC에서 `fopen`을 사용하면 보안 경고가 나올 수 있으며, 이 경우 `#define _CRT_SECURE_NO_WARNINGS`를 파일 맨 위에 추가하면 됩니다.

### 6.3 전처리기(#define) 작성 시 주의사항 요약

1. **상수 매크로**에는 끝에 `;`를 붙이지 않습니다.
   - 올바른 예: `#define MAX 100`
   - 잘못된 예: `#define MAX 100;` → 치환 시 `100;`이 들어가서 문법 오류 가능
2. **함수형 매크로**는 **인자와 전체를 괄호**로 감쌉니다.
   - 올바른 예: `#define Mul(a, b) ((a)*(b))`
   - 잘못된 예: `#define Mul(a, b) (a*b)` → 연산자 우선순위 오류 가능
3. **부작용이 있는 인자**(`++`, `--`, 함수 호출 등)를 매크로에 넣으면 **여러 번 평가**될 수 있으므로 주의합니다.

### 6.4 구조체 접근 연산자 비교

| 상황 | 연산자 | 예시 |
|------|--------|------|
| 구조체 **변수**로 접근 | `.` (점) | `data.n = 10;` |
| 구조체 **포인터**로 접근 | `->` (화살표) | `pd->n = 10;` |

- `pd->n`은 `(*pd).n`과 같은 의미이지만, `->` 연산자가 더 간결하고 자주 사용됩니다.

| 주제 | 내용 |
|------|------|
| 파일 열기 | `fopen_s(&pf, "파일명", "모드")` — `"wt"` 쓰기, `"rt"` 읽기 |
| 파일 쓰기 | `fprintf(pf, "서식", 값들)` |
| 파일 읽기 (줄 단위) | `fgets(buf, 크기, pf)` + `sscanf_s(buf, ...)` |
| 파일 읽기 (직접) | `fscanf_s(pf, "서식", ...)` |
| 파일 닫기 | `fclose(pf)` — 열었으면 반드시 닫기 |
| 상수 매크로 | `#define MAX 100` — 이름을 값으로 치환 |
| 함수형 매크로 | `#define Mul(a, b) ((a)*(b))` — 반드시 **괄호** 사용 |
| 매크로 vs 함수 | 매크로는 텍스트 치환, 함수는 인자를 계산 후 전달 |
| 매크로 부작용 | `++`, `--` 등을 매크로 인자에 넣으면 위험 |
| 삼항 연산자 | `조건 ? 참일_때_값 : 거짓일_때_값` |
| 조건부 컴파일 | `#ifdef`, `#else`, `#endif` — 선택적 컴파일 |
| 구조체 정의 | `struct 이름 { 멤버들; };` |
| 구조체 포인터 접근 | `pd->멤버` (화살표 연산자) |
| 구조체 배열 | `Data darr[5]` — 반복문으로 활용 |
