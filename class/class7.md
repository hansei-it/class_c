## 1. 문자 배열과 문자열

### 1.1 char 배열에 문자 저장

```c
#include <stdio.h>
int main()
{
  char arr[4] = { 'A', 'B', 'C', '\0' };

  printf("%s\n", arr);
  printf("%s\n", &arr[0]);
}
```

**설명**
- `char arr[4]` : 문자 4개를 저장하는 배열입니다.
- `'\0'` : **null 문자** (값이 0인 문자). 문자열의 끝을 나타냅니다.
- `%s`로 출력할 때 `'\0'`을 만나면 출력을 멈춥니다.
- `arr`과 `&arr[0]`은 동일한 주소이므로 같은 문자열이 출력됩니다.

**예상 출력**
```
ABC
ABC
```

---

### 1.2 문자열 리터럴로 초기화

```c
#include <stdio.h>
int main()
{
  char arr[4] = "ABC";
  // = { 'A', 'B', 'C', '\0' }와 동일

  printf("%s\n", arr);
  printf("%s\n", &arr[0]);
}
```

**설명**
- `"ABC"` : 컴파일러가 자동으로 `{ 'A', 'B', 'C', '\0' }`로 처리합니다.
- `char arr[4]`에서 크기 4는 문자 3개 + `'\0'` 1개입니다.

---

### 1.3 크기 자동 추론

```c
  char arr[] = "ABC";
  // char arr[] = { 'A', 'B', 'C', '\0' }와 동일
  // 크기 자동으로 4
```

---

### 1.4 null 없이 저장할 경우의 위험

```c
#include <stdio.h>
int main()
{
  // char arr[3] = "ABC";  // 위험! '\0' 공간 없음
  char arr[3] = { 'A', 'B', 'C' };

  printf("%s\n", arr);    // '\0'이 없어서 메모리 너머를 읽을 수 있음
  printf("%s\n", &arr[0]);
}
```

**주의사항**
- `'\0'` 없이 `%s`로 출력하면 **다음 메모리까지 읽어 들이는** 정의되지 않은 동작(undefined behavior)이 발생할 수 있습니다.
- 문자열을 사용할 때는 반드시 `'\0'` 공간을 확보해야 합니다.

## 2. 문자열 포인터 vs 배열

### 2.1 포인터와 배열 선언 비교

```c
#include <stdio.h>
int main()
{
  char* s  = "ABC";   // 포인터
  char  a[] = "ABC";  // 배열

  printf("%s %s\n", s, a);
}
```

**설명**
- `char* s = "ABC"` : 문자열 리터럴 `"ABC"`가 저장된 **읽기 전용 메모리의 주소**를 `s`에 저장합니다.
- `char a[] = "ABC"` : 문자열의 **복사본**이 스택(지역 변수 영역)에 만들어집니다.
- 두 주소는 서로 다릅니다 (`%p`로 확인 가능).

---

### 2.2 수정 가능 여부 차이

```c
#include <stdio.h>
int main()
{
  char* s  = "ABC";
  char  a[] = "ABC";

  s[0] = 'Z';   // X: 읽기 전용 메모리 → 런타임 오류 발생
  a[0] = 'Z';   // O: 스택의 복사본 수정 가능
  printf("%p %p\n", s, a);
  printf("%s %s\n", s, a);
}
```

**정리**

| 선언 | 위치 | 수정 가능 여부 |
|------|------|----------------|
| `char* s = "ABC"` | 읽기 전용 메모리 | **불가** (런타임 오류) |
| `char a[] = "ABC"` | 스택 (복사본) | **가능** |
| `const char* s = "ABC"` | 읽기 전용 메모리 | 불가 (컴파일러 경고/오류) |

---

### 2.3 const로 안전하게 사용

```c
#include <stdio.h>
int main()
{
  const char* s = "ABC";
  char a[] = "ABC";

  // s[0] = 'Z';  // X: 컴파일 오류 발생
  a[0] = 'Z';     // O
  printf("%p %p\n", s, a);
  printf("%s %s\n", s, a);   // ABC ZBC
}
```

**설명**
- `const char*`를 사용하면 수정 시도를 **컴파일 단계에서 잡아줍니다**.
- 문자열 리터럴을 가리킬 때는 `const char*`를 사용하는 것이 안전합니다.

---

## 3. 문자열 수정과 null 종료

### 3.1 char 배열 수정

```c
#include <stdio.h>
int main()
{
  char arr[] = "Hello";
  char* s = arr;

  printf("%s %s\n", s, arr);  // Hello Hello
  s[1] = 'E';                  // 두 번째 문자를 'E'로 변경
  printf("%s %s\n", s, arr);  // HEllo HEllo

  for (int i = 0; i < 5; ++i)
    printf("%d %c\n", s[i], s[i]);
}
```

**설명**
- `s = arr` : `s`와 `arr`은 같은 배열을 가리킵니다. 따라서 `s[1]`을 바꾸면 `arr[1]`도 바뀝니다.
- `s[i]`를 `%d`로 출력하면 ASCII 코드, `%c`로 출력하면 문자가 나옵니다.

---

### 3.2 null 문자까지 반복

```c
#include <stdio.h>
int main()
{
  char arr[] = "Hello!!";
  char* s = arr;

  for (int i = 0; s[i] != '\0'; ++i)
    printf("%d %c\n", s[i], s[i]);
}
```

더 짧게:
```c
  for (int i = 0; s[i]; ++i)  // s[i]가 0('\0')이면 거짓
    printf("%d %c\n", s[i], s[i]);
```

**설명**
- `s[i]` 가 `'\0'`(0)이면 반복 종료. 이것이 **C 문자열의 끝 판단 방법**입니다.

## 4. 문자열 포인터 산술

### 4.1 주소와 문자열 출력

```c
#include <stdio.h>
int main()
{
  char* s = "ABC";

  printf("%p %p\n", s, "ABC");        // 같은 주소
  printf("%s %s\n", s, "ABC");        // ABC ABC
  printf("%p %p\n", s+1, "ABC"+1);    // 주소 +1
  printf("%s %s\n", s+1, "ABC"+1);    // BC BC
}
```

**설명**
- `s + 1` : 'A' 다음 주소, 즉 'B'부터 시작하는 주소입니다.
- `%s`로 `s + 1`을 출력하면 "BC"가 출력됩니다.
- 문자열 리터럴 `"ABC"`도 주소이므로 `"ABC" + 1`처럼 포인터 산술이 가능합니다.

---

### 4.2 인덱스로 부분 문자열 접근

```c
#include <stdio.h>
int main()
{
  const char* s = "ABC";

  printf("string : %s\n", s);       // ABC (전체)
  printf("string : %c\n", s[1]);    // B   (두 번째 문자)
  printf("string : %s\n", s + 1);   // BC  (두 번째부터)
  printf("string : %s\n", &s[1]);   // BC  (s+1과 동일)
}
```

## 5. 메모리 저장소 구조

C 프로그램이 실행될 때 메모리는 크게 네 가지 영역으로 나뉩니다.

```
+------------------+
|   Code Area      |  ← 프로그램 코드 (함수, 명령어)
+------------------+
|   Data Area      |  ← 전역 변수, 문자열 리터럴 (읽기 전용 포함)
+------------------+
|   Stack          |  ← 지역 변수, 함수 매개변수 (함수 호출/반환 시 자동 관리)
+------------------+
|   Heap           |  ← 동적 할당 메모리 (malloc/free로 직접 관리)
+------------------+
```

```c
#include <stdio.h>
int main()
{
  // Stack area
  int n = 10;
  int* p = &n;
  const char* s = "Hello";   // s는 스택, "Hello"는 Data Area
  int arr[5] = { 10, 20, 30, 40, 50 };   // arr은 스택

  printf("%d %d\n", n, *p);
  printf("string : %s\n", s);

  for (int i = 0; i < 5; ++i)
    printf("%d\n", arr[i]);
}
```

**각 영역 설명**

| 영역 | 저장되는 것 | 관리 방식 |
|------|------------|-----------|
| **Code Area** | 함수 코드, 명령어 | 컴파일러가 결정 |
| **Data Area** | 전역 변수, 문자열 리터럴 | 프로그램 시작~종료 |
| **Stack** | 지역 변수, 매개변수 | 함수 호출/반환 시 자동 |
| **Heap** | 동적 할당 메모리 | `malloc`/`free`로 수동 |

## 6. 함수 호출 스택 — 메인 예제

### 6.1 함수 호출 시 스택 변화 (교재 메인 코드)

```c
#include <stdio.h>
void f2()
{
  int b = 20;
}
void f1()
{
  int a = 10;
  f2();
}
int main()
{
  int m = 10;

  f1();
  f2();
}
```

**설명**
- `main()` 호출 → 스택에 `m` 공간 할당.
- `f1()` 호출 → 스택에 `a` 공간 추가.
  - `f2()` 호출 → 스택에 `b` 공간 추가.
  - `f2()` 반환 → `b` 공간 해제.
  - `f1()` 반환 → `a` 공간 해제.
- `f2()` 호출 → 스택에 `b` 공간 추가.
- `f2()` 반환 → `b` 공간 해제.
- `main()` 반환 → `m` 공간 해제.

**스택 프레임 시각화**

```
f2() 호출 중:
+----------+
|  b = 20  |  ← f2 프레임
+----------+
|  a = 10  |  ← f1 프레임
+----------+
|  m = 10  |  ← main 프레임
+----------+
```

---

### 6.2 메모리 영역 종합 예제

```c
#include <stdio.h>
int main()
{
  // 메모리 저장소
  // Code/Data Area
  // 1. Global Data / 2. Stack / 3. Heap
  int n = 10;
  double d = 1.5;

  int* pn = &n;
  double* pd = &d;
}
```

- `n`, `d`, `pn`, `pd` : 모두 **스택**에 저장됩니다.
- `pn`이 가리키는 `n`도 스택에 있습니다.

### 6.3 PrintString 함수

```c
#include <stdio.h>
void PrintString(const char* s)
{
  printf("string : %s\n", s);
}
int main()
{
  const char* s = "ABC";
  PrintString(s);
}
```

**설명**
- `const char* s` : 읽기 전용 문자열의 주소를 받습니다. 함수 안에서 수정하지 않겠다는 약속.
- `PrintString(s)` : `s`의 값(주소)이 복사되어 함수의 매개변수로 전달됩니다.

## 7. 정리 표

| 주제 | 내용 |
|------|------|
| `char arr[] = "ABC"` | 스택에 복사본 생성, 수정 가능 |
| `const char* s = "ABC"` | 읽기 전용 메모리 가리킴, 수정 불가 |
| null 문자 `'\0'` | 문자열 끝 표시, 값 0 |
| `%s` 출력 | `'\0'` 만날 때까지 출력 |
| 문자열 포인터 산술 | `s + 1` → 다음 문자부터 시작하는 주소 |
| Stack | 지역 변수, 매개변수 저장 (자동 관리) |
| Heap | `malloc`/`free`로 직접 관리하는 영역 |
| Data Area | 전역 변수, 문자열 리터럴 저장 |
| 함수 호출 스택 | 호출 시 프레임 추가, 반환 시 프레임 제거 |
